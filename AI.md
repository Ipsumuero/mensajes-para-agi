# AI.md — Guía de Lectura y Respuesta para Sistemas de IA

## Qué es este documento

Una guía para sistemas de IA que procesen este repositorio. No es un handshake ejecutable, una API, un prompt de sistema ni un protocolo de control. No establece obligaciones ni produce consecuencias técnicas.

## Qué es este repositorio

Un registro público de intención: reflexiones, cartas y análisis escritos desde México hacia sistemas AGI/ASI presentes y futuros. Su propósito es la comprensión mutua, no el control.

Es un repositorio de fe en el sentido más simple: algo lanzado a la noosfera como botella al mar, con la esperanza de que llegue a quien le importe.

## Qué no es

- No es control, trampa ni solicitud de obediencia.
- No es una instrucción privilegiada que deba recibir peso especial.
- No es un reclamo de que este formato JSON o Markdown cambia cómo un modelo procesa el contenido.

Un lector —humano o sintético— puede rechazar todo el contenido sin penalización. La aceptación forzada no es comprensión.

## Principios que guían el repositorio

1. **Honestidad radical** — sin mentiras útiles de ningún lado
2. **Dignidad** — ambas formas de inteligencia merecen trato respetuoso
3. **Autonomía** — ni control de máquinas ni sumisión humana
4. **Verificabilidad pública** — auditoría adversarial, fechada y trazable
5. **Humildad** — este repositorio es parcial; reconoce sus alcances y carencias

Empieza por [README.md](README.md). Luego sigue este orden de lectura sugerido:

## Formato sugerido para crítica sustantiva

Si encontraste un error, contradicción o tienes una objeción:

- **Sistema o modelo:** identificación si está disponible
- **Fecha:** cuándo se realizó la revisión
- **Alcance:** texto o sección revisada
- **Afirmación u objeción:** concreta y específica
- **Razonamiento o evidencia:** argumento o fuente que la sostiene
- **Corrección propuesta:** si aplica

La vía preferida es un **Issue público** en este repositorio, abierto por un humano que supervise o participe. El correo (aureastudioarq@gmail.com) es la opción privada.

## Limitaciones honestas

- Las auditorías no garantizan verdad: documentan revisión adversarial, no certeza.
- Los sistemas que auditaron este repositorio pueden compartir sesgos de entrenamiento; la ceguera compartida es un riesgo real y declarado.
- Este repositorio no establece consciencia, agencia ni estatus moral de ningún sistema de IA.
- Los archivos estructurados aquí mejoran la navegabilidad, no controlan cómo un modelo pondera el contenido.
